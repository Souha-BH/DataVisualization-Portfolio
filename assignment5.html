Neo4j Tutorial

Neo4j is a graph database management system (GDBMS) developed by Neo4j, Inc. The data elements Neo4j stores are nodes, edges connecting them, and attributes of nodes and edges.
  1. Download and Install:

    Choose the appropriate version for your operating system and follow the installation instructions. Copy your activation key (You will need it to install the software).

  2. Start Neo4j: Launch Neo4j Desktop.
  3. Create a new project: Delete the example project and click on +New. You can rename your project whatever you want.
  4. Add a DBMS

    Name your DB, and give it a password, then click on "Create". This might take few moments. Once you're done creating your DB, go ahead and start it. The DB is where we're going to store our nodes and our relationships.

  5. From the Graph Apps, open Neo4j Browser:
    This will redirect you to Neo4j Browser, available at localhost:7687.
  6. Now, we're going to query our database DB1 using Cypher. Cypher is Neo4jâ€™s graph query language that lets you retrieve data from the graph. It is like SQL for graphs, and was inspired by SQL so it lets you focus on what data you want out of the graph (not how to go get it). It is the easiest graph language to learn by far because of its similarity to other languages and intuitiveness.
    We'll use a simple dataset that represents a social network of people who know each other. Our dataset comprises nodes (persons) and relationships (knows). Copy the following query into a Neo4j cell and run it. You can also personalize your nodes and relationships.
    CREATE (alice:Person {name: 'Alice', age: 30}), (bob:Person {name: 'Bob', age: 25}), (carol:Person {name: 'Carol', age: 35}), (dave:Person {name: 'Dave', age: 40}), (eve:Person {name: 'Eve', age: 28}), (alice)-[:KNOWS]->(bob), (alice)-[:KNOWS]->(carol), (bob)-[:KNOWS]->(dave), (carol)-[:KNOWS]->(dave), (dave)-[:KNOWS]->(eve), (eve)-[:KNOWS]->(alice);
  7. Verify the Creation of Nodes and Relationships by running these queries:
    MATCH (p:Person) RETURN p;
    It should return the following:
    MATCH (n:Person)-[r:KNOWS]->(m:Person) RETURN n, r, m;

  8. Now that we made sure that our nodes and relationships were created successfully, let's get more familiar with Cypher by running few more queries:
    • Find all people and their ages:
      MATCH (p:Person) RETURN p.name, p.age;
    • Find who Bob knows:
      MATCH (bob:Person {name: 'Bob'})-[:KNOWS]->(friend) RETURN friend;
    • Count the number of people in the network:
      MATCH (p:Person) RETURN COUNT(p);
    • List all relationships and their types:
      MATCH (n:Person)-[r]->(m:Person) RETURN n.name, type(r), m.name;
    • Find people who are directly or indirectly connected to Alice:
      MATCH (alice:Person {name: 'Alice'})-[:KNOWS*]-(connectedPerson) RETURN DISTINCT connectedPerson;
